## The Paradigm Shift: Numbers Are Already Polynomial Cosets

The old framing treats irrational numbers as *limit processes* — sequences of rationals converging to something unreachable. That's the analyst's view, and it's exactly wrong for computation. The algebraist's view is different and far more useful:

**An irrational algebraic number is not an approximation target. It is an exact coset in a quotient ring.**

### Three manifestations of "number as polynomial"

**1. Positional notation is polynomial evaluation.**

The binary string `1011` encodes the polynomial $p(x) = x^3 + x + 1 \in \mathbb{Z}[x]$, evaluated at $x = 2$. Decimal `1011` is the same polynomial evaluated at $x = 10$. Hex `0x1B` is evaluation at $x = 16$. The base is not fundamental — the polynomial is. The *ring* $\mathbb{Z}[x]$ contains all of them simultaneously as formal objects.

**2. Galois fields are polynomial arithmetic modulo an irreducible.**

$\text{GF}(2^8) = \mathbb{F}_2[x] / (x^8 + x^4 + x^3 + x + 1)$

A byte `0xAB` is the polynomial $x^7 + x^3 + x + 1$ over $\mathbb{F}_2$. Addition is XOR. Multiplication is polynomial multiplication reduced modulo the irreducible. Every element is exact. AES field multiplication has **zero** approximation because the arithmetic stays inside the quotient ring. This is not a trick — it's the natural mathematics.

**3. Algebraic irrationals are quotient ring elements — the same construction.**

$\sqrt{2}$ is a root of $x^2 - 2$, which is irreducible over $\mathbb{Q}$. Therefore:

$$\mathbb{Q}(\sqrt{2}) \cong \mathbb{Q}[x]/(x^2 - 2)$$

Elements are polynomials of degree $< 2$ with rational coefficients: $a + b\sqrt{2}$. The rule $(\sqrt{2})^2 = 2$ is exactly what polynomial multiplication mod $x^2 - 2$ gives: $x \cdot x \equiv 2 \pmod{x^2-2}$. This is **the same construction** as $\text{GF}(2^8)$, just over $\mathbb{Q}$ instead of $\mathbb{F}_2$.

The machine already does exact polynomial-coset arithmetic for $\text{GF}(2^8)$ in hardware. The only question is why we don't do it for $\mathbb{Q}(\sqrt{2})$.

### Why floating-point is the wrong abstraction

IEEE 754 `double` is not a representation of $\mathbb{R}$. It is a representation of **a finite subset of $\mathbb{Q}$**, specifically dyadic rationals with 53-bit mantissas. When you write `double x = sqrt(2.0)`, you project $\sqrt{2} \in \mathbb{Q}(\sqrt{2})$ down into a nearby rational, discarding the algebraic identity entirely.

The machine that receives `x = 1.41421356237...` has no idea it came from $\mathbb{Q}(\sqrt{2})$. It cannot know that $x^2$ should equal *exactly* 2. It computes a float slightly off from 2.0, propagates the error, and the type system offers no recourse.

Continued fractions partially address the approximation accuracy problem. They do not address the *identity problem*. You can converge to $\sqrt{2}$ arbitrarily closely with rationals, but no rational *is* $\sqrt{2}$, and the downstream algorithm can't detect the difference.

### Galois theory: symmetries give exact invariants

The Galois group $\text{Gal}(\mathbb{Q}(\sqrt{2})/\mathbb{Q}) \cong \mathbb{Z}/2\mathbb{Z}$ is generated by the automorphism:
$$\sigma: \sqrt{2} \mapsto -\sqrt{2}$$

From this single symmetry we get two invariants that are *provably in* $\mathbb{Q}$ and computable exactly:

$$\text{Norm}(a + b\sqrt{2}) = (a+b\sqrt{2})(a-b\sqrt{2}) = a^2 - 2b^2 \in \mathbb{Q}$$
$$\text{Trace}(a + b\sqrt{2}) = (a+b\sqrt{2}) + (a-b\sqrt{2}) = 2a \in \mathbb{Q}$$

The minimal polynomial of any element over $\mathbb{Q}$ is:
$$x^2 - \text{Tr}(\alpha)\cdot x + N(\alpha)$$

These hold exactly. The inverse of $\alpha$ is $\sigma(\alpha)/N(\alpha)$ — field division reduces to computing a rational. No Newton iteration. No continued fractions. No ULP budget.

For nested extensions: $\mathbb{Q}(\sqrt{2}, \sqrt{3})$ is a degree-4 extension where elements are $a_0 + a_1\sqrt{2} + b_0\sqrt{3} + b_1\sqrt{6}$, represented as pairs over $\mathbb{Q}(\sqrt{2})$. Composition is recursive and exact at every level.

All assertions pass. `sqrt(2.0)*sqrt(2.0) == 2.0` is `FALSE` as expected — that ULP error is real.

## The Conceptual Architecture

### The single unifying observation

A number in base $\beta$ is a polynomial in $\mathbb{Z}[x]$ evaluated at $x = \beta$. That's not a metaphor — it's the literal definition of positional notation. When you write `0xAB` in a GF(2⁸) context, that byte *is* the polynomial $x^7 + x^3 + x + 1$ over $\mathbb{F}_2$, and multiplication is polynomial multiplication mod the AES irreducible. When you write $\sqrt{2}$, it *is* the coset $\bar{x}$ in $\mathbb{Q}[x]/(x^2 - 2)$. Same construction, different coefficient ring, different modulus. The machine already does the former exactly in hardware — the question is why we don't do the latter.

### What floating-point actually is

`double` is not a subset of $\mathbb{R}$. It's a subset of $\mathbb{Q}$ — specifically dyadic rationals with 53-bit mantissas, i.e. the ring $\mathbb{Z}[1/2]$ with bounded precision. When you compute `sqrt(2.0)`, you *project* the element $\sqrt{2} \in \mathbb{Q}(\sqrt{2})$ into a nearby element of $\mathbb{Z}[1/2]$, discarding the algebraic identity $(\sqrt{2})^2 = 2$ in the process. The run outputs this plainly: `sqrt(2.0)*sqrt(2.0) = 2.0000000000000004`. The downstream algorithm receives `2.0000000000000004` with no way to know it should have been exactly 2.

### The type system encoding

`SqrtExt<D, Base>` encodes the field $\text{Base}(\sqrt{D}) \cong \text{Base}[x]/(x^2 - D)$. Three things fall out for free:

**The product rule is the minimal polynomial.** Multiplication is defined by the single relation $(\sqrt{D})^2 = D$, which is polynomial multiplication mod $x^2 - D$. There's no approximation to define — it's a rewrite rule.

**Division is a closed-form norm computation.** In $\mathbb{Q}(\sqrt{D})$: $\alpha^{-1} = \sigma(\alpha)/N(\alpha)$ where $\sigma(a+b\sqrt{D}) = a - b\sqrt{D}$ is the Galois conjugate and $N(\alpha) = a^2 - Db^2 \in \mathbb{Q}$. Field division reduces to rational arithmetic. No iteration, no Newton-Raphson, no ULP budget.

**Nesting is recursive and exact.** `SqrtExt<3, SqrtExt<2>>` is $\mathbb{Q}(\sqrt{2}, \sqrt{3})$ — a degree-4 extension with basis $\{1, \sqrt{2}, \sqrt{3}, \sqrt{6}\}$. The outer level calls $\mathbb{Q}(\sqrt{2})$ arithmetic at every step, which calls $\mathbb{Q}$ arithmetic. Each level is exact. The compiler statically prevents you from adding a `Q2` element to a `Q3` element without explicit embedding — the mixed-field accident that float permits silently.

### What continued fractions actually are in this framework

Demo 7 proves something the original explanation missed entirely: continued fraction convergents $p/q$ for $\sqrt{D}$ are not a separate theory. They are the solutions to the **Pell equation** $p^2 - Dq^2 = \pm 1$ — which is precisely $|N(p - q\sqrt{D})| = 1$. The norm is the Pell equation. Continued fractions are the orbit of the Galois conjugation $\sigma: \sqrt{D} \mapsto -\sqrt{D}$ acting on the integer lattice $\mathbb{Z}[\sqrt{D}]$. They converge to $\sqrt{D}$ because each convergent is a unit in $\mathbb{Z}[\sqrt{D}]$ — the closest rational approach to the irrational coset while remaining in the integer sublattice. Galois theory and continued fractions are not "related" — they are the same object seen from different angles.

### What this framework cannot do

The cubic extension $\mathbb{Q}(\sqrt[3]{2})$ is exact for arithmetic, but is **not Galois** over $\mathbb{Q}$ — the two complex roots $\omega\sqrt[3]{2}$ and $\omega^2\sqrt[3]{2}$ lie outside the field. The Galois closure requires $\mathbb{Q}(\sqrt[3]{2}, \omega)$ with $\text{Gal} \cong S_3$. The `CubicExt` type handles the real embedding exactly, but the full Galois group isn't representable without also adjoining $\omega \in \mathbb{Q}(\sqrt{-3})$. Transcendental numbers ($\pi$, $e$) have no minimal polynomial at all and genuinely require approximation — this framework is for *algebraic* numbers only.
